## Reporting Vulnerabilities

**DO NOT** create public issues for security vulnerabilities.

### Private Vulnerability Reporting (Recommended)

This repository has [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) enabled.

**To report a vulnerability:**

1. Go to the [Security tab](https://github.com/utsavtulsyan/dart-unused-code/security)
2. Click "Report a vulnerability"
3. Fill in the advisory form with:
   - Title and description
   - Affected versions
   - Severity assessment
   - Reproduction steps
   - Impact details
   - Suggested fix (optional)

GitHub will notify the maintainers privately, and you'll receive credit for the report.

### Alternative: Email Report

If you prefer, you can also email: **support@utsavtulsyan.com**

Include: description, reproduction steps, impact, suggested fix (optional)

**Response time:** Within 48 hours.

## Privacy

This extension:
- Runs 100% locally
- No network requests
- No data collection
- Only reads Dart files in workspace
- No arbitrary code execution

Security fixes released promptly via GitHub and VS Code Marketplace.
